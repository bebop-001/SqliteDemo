This app supports some SQL regular expressions and also allows you to select integers.

  • You can search on two values: NAME which is a String and AGE which is an Int.
  • Boolean: AND, OR, NOT: conditions can be connected using these values. For example, "NOT AGE = 30" will find all cards for which the customer's age is not 30. These operators do have a precedence of evaluation, so use parentheses '()' if you have any doubt.
  • ESCAPE: This clause allows you to say the next character is not a meta-character used for the search. To say that the dollar sign ($) is an escape character you would say "ESCAPE '$'"
  • GLOB and LIKE: Both of these operators support regex-like expressions. Hopefully they will do what is needed. REGEXP and MATCH are "application-defined" functions.
    • LIKE: LIKE supports two meta characters:"%" and "_"
      • %: A percent symbol ("%") in the LIKE pattern matches any sequence of zero or more characters.
      • An underscore ("_") in the LIKE pattern matches any single character.
    • GLOB: Like "LIKE" but with unix style wild cards:
      • *: 0 or more of any character
      • ?: 1 of any character.
      • [a-n]: any character between a and n inclusive.
      • [a,Z]: an 'a' or a 'Z'.
  • EXAMPLES:
    NOTE: (') is significant. 'xyz' denotes s string containing xyz. Also, you can turn on case using a pragma, but the examples below are case-insensitive.'
    • first character of name is a 'j':
      NAME LIKE 'j%'
    • Int fields are cast to strings for this test so to find any customer with a '2' in their age, you can use:
      AGE LIKE '%2%'
    • first character of name is a 'j' and last is e:
      NAME LIKE 'j%e'
    • first character of name is a 'j' and contains an e:
      NAME LIKE 'j%e%'
    • first character of name is a 'j' and name does not contain an 's':
      NAME LIKE 'j%e%' AND NOT NAME LIKE '%s%'
    • Name contains a '%'. Use '#' to excape the '%' meta-char.
      NAME LIKE '%#%%' ESCAPE '%'
    • Name starts with a lower-case 'j':
      NAME GLOB 'j*'
    • Name starts with a upper-case 'J':
      NAME GLOB 'J*'
    • Name starts with a 'j' or a 'J':
      NAME GLOB '[Jj]*'
